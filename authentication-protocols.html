Authentication Protocols

What’s OAuth 2.0?

OAuth is a standard for using tokens when you need to 1). get permission to bring in a user’s data from another system - for example you want to use a user’s Facebook or Google data, or 2). When you have a mobile application, it’s more secure to use an access token for authentication rather than storing a user’s login credentials on their mobile device.

To be clear, there is OAuth 1.0a and OAuth2, which are very different from each other. OAuth2 is more widely used, so we’ll focus on that, referring to it as OAuth.

OAuth has 4 primary grant types: password credentials, client credentials, authorization and implicit.

The password grant type gives the ability to get an access token with a given username and password. The password grant type is used when you have built your own OAuth service and application and you want to authenticate users for your mobile apps.

Client credentials grant type is used when you’re building an application that is doing non-user related tasks.

Authorization grant type is when you want to bring in user data from another system on a server side app, and implicit grant type is similar but for a client-side app.
