Security & Compliance

Building out a secure authentication and authorization solution usually requires a team of security experts, as well as a team that can manage and update security protocols as they evolve. Companies that fail to update a protocol or enhance their security best practices are prime targets for security breaches. What was secure yesterday is probably not secure today, so make sure you are continuously revisiting and reiterating your security strategy. Also, sorry to break it to you, but the internet is not a safe place to learn about security. It’s a great start, but if you are relying on the internet, you will hear conflicting information.

Here are a list of security protocols and requirements you will want to familiarize yourself with in order to build a secure authentication and authorization solution for your business.

User Level Security

Authorization

Ensure your strategy includes authorization! Most often companies think about authorization after authentication, when the two go hand-in-hand. Ensure your solution can handle all the different roles, permissions, groups for your organization; and can be changed easily on the fly. The last thing you want is to get stuck using a model that isn’t flexible or agile.

Encrypt all data in-transit and at rest in 256-bit AES encryption

Anything less is not worth the investment and risk!

Ensure you have the most up to date Account/Email verification workflows

Multi- Factor Authentication

A method in which a user is only granted access after successfully presenting several separate pieces of evidence to an authentication mechanism.

Use Verified Identity Standards

LDAP, SAML 2.0, OAuth 2.0, OpenID, OpenID Connect, JSON Web Tokens (JWTs), WS-Federation, Shibboleth, OpenAthens, NAPPS, XACML, Kerberos, CAS

Single Sign-On with JWT

All single sign-on performed should use a secure and signed JSON Web Token (JWT) to prevent unauthorized access or tampering of authentication and user data between applications.

We won’t go too much in detail here as every company has it’s own security requirements. Here’s a general list of requirements:
